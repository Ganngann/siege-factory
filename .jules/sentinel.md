## 2025-02-23 - Path Traversal in Modding System
**Vulnerability:** The `ModRegistry` methods for loading files (like `load_data`, `load_all_data`, `load_texture`, `load_story`) were vulnerable to path traversal because they blindly concatenated user-provided filenames with the mod's base path (`path.join("data").join(filename)`). This allowed attackers to use patterns like `../` to read files outside the designated `mods/` directory.
**Learning:** In Rust's `std::path::PathBuf::join()`, appending an absolute path replaces the base path entirely. Additionally, normal path manipulation doesn't automatically prevent `..` components from navigating up the directory tree. Because mods often provide data via these filename lookup functions, it presents a serious file-disclosure risk if filenames are not sanitized.
**Prevention:** Always validate user-provided or mod-provided file paths before joining them with base directories. A robust check should use `Path::new(filename)` to reject absolute paths and iterate through `path.components()` to ensure all components are `Component::Normal`. Furthermore, to prevent subtle platform-specific bypasses, explicitly reject paths containing `..` or backslashes `\`.
